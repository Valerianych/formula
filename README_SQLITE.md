# SQLite-версия F1 AI Analyst

В проект добавлена локальная SQLite-база данных. Теперь сайт может брать данные не напрямую из внешних API, а из файла `data/f1.sqlite`.

## Что изменилось

- Добавлен новый сервер `server-sqlite.ts`.
- Добавлен слой базы данных `src/db/f1Sqlite.ts`.
- Добавлены API-маршруты `src/db/f1Routes.ts`.
- Добавлен скрипт заполнения базы `scripts/seed-db.ts`.
- Команды `npm run dev`, `npm run build`, `npm run start` переключены на SQLite-сервер.

## Как запустить

```bash
npm install
npm run db:seed
npm run dev
```

После запуска сайт откроется через локальный сервер, а данные будут браться из SQLite.

## Как пересоздать базу

```bash
npm run db:reset
```

Команда удаляет старые таблицы и заново заполняет базу данными.

## Где лежит база

По умолчанию база создаётся здесь:

```text
data/f1.sqlite
```

Файл базы не добавляется в GitHub, потому что он генерируется автоматически.

## Какие данные лежат в базе

В SQLite создаются таблицы:

- `sessions` — сезоны, Гран-при и сессии;
- `session_drivers` — пилоты в конкретной сессии;
- `laps` — круги пилотов и сектора;
- `weather` — погодные условия;
- `race_events` — события гонки;
- `driver_standings` — личный зачёт пилотов;
- `constructor_standings` — зачёт команд;
- `race_results` — результаты гонок.

## Как проверить, что база работает

После запуска открыть в браузере:

```text
http://localhost:3000/api/db-status
```

Если всё нормально, сервер покажет количество записей в таблицах.

## Как это объяснять на защите

В проекте используется локальная SQLite-база данных. В неё предварительно загружаются сведения о сезонах, гонках, пилотах, кругах, погоде, событиях и турнирных таблицах. Сервер Express получает данные из SQLite и отдаёт их React-интерфейсу через REST API. Такой подход снижает зависимость от внешних сервисов и позволяет приложению работать даже при недоступности сторонних API.
