# Данные Формулы-1 на Vercel

Проект переведён в режим API-only: на странице больше не должен использоваться старый экран с моками и резервными значениями.

## Какие источники используются

- `OpenF1 API` — сессии, пилоты, погода, сообщения Race Control, круги пилотов.
- `Jolpica API` — таблицы чемпионата, личный зачёт, кубок конструкторов, победители гонок.

## Как это работает

Страница загружает файл:

```text
public/f1-visual-fixes.js
```

Этот файл строит интерфейс заново и запрашивает данные только через API-маршруты Vercel:

```text
/api/sessions
/api/session-data
/api/driver-laps
/api/standings
/api/results
/api/analyze
/api/db-status
```

Если внешний API не вернул данные, на сайте показывается пустое состояние или сообщение об ошибке. Выдуманные данные, старые fallback-таблицы и мок-телеметрия не должны использоваться.

## Как проверить на Vercel

После деплоя открыть:

```text
https://адрес-сайта.vercel.app/api/db-status
```

Должно быть указано:

```text
OpenF1 API + Jolpica API
```

Также можно проверить:

```text
/api/results?year=2024
/api/standings?year=2024
/api/sessions?year=2024
```

## Как объяснять на защите

В проекте используются внешние открытые API Формулы-1. OpenF1 отдаёт данные по сессиям, пилотам, погоде и кругам. Jolpica отдаёт результаты гонок, победителей этапов и турнирные таблицы. Интерфейс сайта обращается к собственным API-маршрутам Vercel, которые получают и подготавливают данные перед выводом на страницу.
