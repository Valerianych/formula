# Данные Формулы-1 на Vercel

Проект настроен так, чтобы данные брались не из демо-файла, а из внешних API.

## Какие источники используются

- `OpenF1 API` — сессии, пилоты, погода, сообщения Race Control, круги пилотов.
- `Jolpica API` — таблицы чемпионата, личный зачёт, кубок конструкторов, победители гонок.

## Как это работает

React-сайт обращается к своим маршрутам:

```text
/api/sessions
/api/session-data
/api/driver-laps
/api/standings
/api/results
/api/analyze
/api/db-status
```

Эти маршруты находятся в папке `api/`. На Vercel они работают как serverless functions. Они запрашивают реальные внешние API и возвращают данные сайту.

## Важно

Данные больше не хранятся во встроенном демо-файле. Если внешний API не ответит, сайт не должен подставлять выдуманные результаты. Вместо этого API вернёт пустой список и сообщение, что внешний источник временно недоступен.

## Как проверить на Vercel

После деплоя открыть:

```text
https://адрес-сайта.vercel.app/api/db-status
```

Должно быть указано:

```text
OpenF1 API + Jolpica API
```

## Как объяснять на защите

В проекте используются внешние открытые API Формулы-1. OpenF1 отдаёт данные по сессиям, пилотам, погоде и кругам. Jolpica отдаёт результаты гонок, победителей этапов и турнирные таблицы. Сайт не обращается к ним напрямую из браузера, а использует собственные API-маршруты на Vercel. Это делает структуру проекта понятнее и позволяет централизованно обрабатывать данные перед выводом на страницу.
