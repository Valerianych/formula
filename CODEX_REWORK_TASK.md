# Задача для Codex: переделать сайт Formula 1 под OpenF1 API

Нужно переделать фронт так, чтобы главным источником данных был backend endpoint:

```txt
GET /api/race-dashboard?session_key={session_key}
```

Этот endpoint уже собирает полный пакет гонки из OpenF1:

- `summary` — общий итог гонки, топ-3, победитель, количество DNF/DNS/DSQ, быстрый круг, пит-стопы, погода.
- `drivers` — список пилотов с командой, номером, стартовой и финишной позицией.
- `driver_summaries` — подробная информация по каждому пилоту: круги, лучший круг, средний круг, пит-стопы, шины, позиции, события Race Control.
- `race_result` — официальный итог сессии из OpenF1.
- `starting_grid` — стартовая решётка, если OpenF1 её вернул.
- `laps` — круги, секторы и скорости.
- `pit_stops` — пит-стопы.
- `stints` — отрезки на шинах.
- `positions` — изменение позиций по ходу гонки.
- `intervals` — интервалы и отрывы, если доступны.
- `track_map` — нормализованные координаты x/y для карты трассы.
- `race_control` / `events` — сообщения Race Control.
- `weather` — погода.
- `overtakes` — обгоны, если доступны.
- `team_radio` — радио, если доступно.
- `issues` — вычисленные проблемные моменты: штрафы, DNF/DNS/DSQ, сильная потеря позиций, медленные пит-стопы, события Race Control.
- `data_quality` — какие блоки реально доступны.

## Важные правила

1. Убрать все моки, фейковые гонки, фейковую телеметрию и симуляции.
2. Если OpenF1 не вернул данные — показывать пустой блок с нормальным текстом, а не выдуманные данные.
3. Не писать, что пилот виноват. Использовать формулировки:
   - “возможный проблемный момент”
   - “Race Control отметил событие”
   - “по данным видно”
4. Главная задача — сайт должен быть понятен человеку, который не шарит в Формуле-1.
5. GigaChat должен получать только краткий контекст гонки, а не огромный массив всех кругов/location.

## Страницы/блоки, которые нужно сделать

### 1. Выбор гонки

Использовать:

```txt
GET /api/sessions?year=2025
```

Показать выбор сезона и гонки. Сессия по умолчанию — Race.

### 2. Итог гонки

Из `/api/race-dashboard` взять:

- `summary.winner`
- `summary.top3`
- `drivers`
- `race_result`

Показать:

- 1 место
- 2 место
- 3 место
- таблицу всех пилотов
- статус DNF/DNS/DSQ
- отставание от лидера
- количество кругов

### 3. Карта гонки

Использовать:

```ts
race.track_map.points
```

Нарисовать SVG-карту. Координаты уже нормализованы от 0 до 100.

Если `track_map.points.length === 0`, показать:

```txt
Карта недоступна: OpenF1 не вернул location-координаты для этой сессии.
```

### 4. Информация о пилоте

При выборе пилота брать объект из `driver_summaries`.

Показать:

- фото
- имя
- команда
- номер
- стартовая позиция
- финишная позиция
- изменение позиции
- лучший круг
- средний круг
- количество пит-стопов
- пит-стопы по кругам
- стратегия шин `stints`
- события Race Control по пилоту

### 5. Пит-стопы

Показать таблицу:

- пилот
- круг
- lane_duration
- stop_duration

Если `stop_duration` нет, подписать:

```txt
OpenF1 вернул только время проезда по пит-лейну.
```

### 6. Шины

Использовать `stints`.

Показать по каждому пилоту:

- compound
- lap_start
- lap_end
- tyre_age_at_start

Добавить расшифровку:

- SOFT — мягкие
- MEDIUM — средние
- HARD — жёсткие
- INTERMEDIATE — для лёгкого дождя
- WET — дождевые

### 7. Сравнение гонщиков

Пользователь выбирает двух пилотов. Сравнить:

- итоговую позицию
- стартовую позицию
- изменение позиции
- лучший круг
- средний круг
- медианный круг
- количество пит-стопов
- самый быстрый пит-стоп
- средний пит-стоп
- количество событий Race Control
- количество проблемных моментов из `issues`

### 8. Кто накосячил / проблемные моменты

Использовать `issues`.

Заголовок лучше сделать:

```txt
Проблемные моменты гонки
```

Не использовать грубое “кто виноват”.

В карточке проблемы показать:

- пилот
- команда
- тип проблемы
- круг, если есть
- severity
- message
- source

### 9. ИИ-чат GigaChat

Использовать:

```txt
POST /api/chat
```

Тело запроса:

```json
{
  "question": "Кто лучше проехал гонку?",
  "messages": [],
  "raceContext": {
    "session": "race.session",
    "summary": "race.summary",
    "top3": "race.summary.top3",
    "issues": "race.issues.slice(0, 30)",
    "driver_summaries": "краткая версия driver_summaries без огромных positions/laps"
  }
}
```

Ответ брать из `answer` или `analysis`.

Если GigaChat недоступен, backend вернёт factual-fallback без выдуманных данных.

## Что убрать с фронта

- Убрать FastF1-симуляцию, если она отображается как реальные данные.
- Убрать текст “OpenF1 Live”, если это не live. Лучше: “OpenF1 Historical Data”.
- Убрать `isDemoData` как источник фейковых данных. Можно оставить только `data_quality.no_mock_data`.
- Убрать любые заранее придуманные победы, круги, погоду, трассы.

## Проверка готовности

Сайт готов, если:

1. Выбор года и гонки работает.
2. Гонка грузится через `/api/race-dashboard`.
3. Показывается топ-3 и таблица результатов.
4. Можно выбрать пилота и увидеть пит-стопы по кругам.
5. Есть карта или честное сообщение, что OpenF1 не вернул location.
6. Есть сравнение двух пилотов.
7. Есть блок проблемных моментов.
8. Есть чат GigaChat.
9. Нет моков и фейковых данных.
